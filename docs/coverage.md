# Покрытие сценариев

Проверка, что TypeSpec-контракт покрывает все сценарии владельца и гостя и все правила
бронирования. Каждая строка — сценарий или правило из [domain.md](./domain.md), сопоставленный
с операцией(ями) спецификации в каталоге [`spec/`](../spec).

## Сценарии владельца

| Сценарий | Операция | Файл | Статус-коды |
|----------|----------|------|-------------|
| Создать тип события (`id`, название, описание, длительность) | `POST /event-types` | [event-types.tsp](../spec/event-types.tsp) | 201 · 409 · 422 |
| Просмотреть предстоящие встречи (все типы, один список) | `GET /bookings` | [bookings.tsp](../spec/bookings.tsp) | 200 |
| (доп.) Профиль владельца | `GET /owner` | [owner.tsp](../spec/owner.tsp) | 200 |

## Сценарии гостя

| Сценарий | Операция | Файл | Статус-коды |
|----------|----------|------|-------------|
| Посмотреть список типов (название, описание, длительность) | `GET /event-types` | [event-types.tsp](../spec/event-types.tsp) | 200 |
| Открыть один тип события | `GET /event-types/{eventTypeId}` | [event-types.tsp](../spec/event-types.tsp) | 200 · 404 |
| Выбрать свободный слот на 14 дней | `GET /event-types/{eventTypeId}/slots` | [slots.tsp](../spec/slots.tsp) | 200 · 404 |
| Создать бронирование на слот (имя + email) | `POST /bookings` | [bookings.tsp](../spec/bookings.tsp) | 201 · 404 · 409 · 422 |
| (доп.) Подтверждение брони | `GET /bookings/{bookingId}` | [bookings.tsp](../spec/bookings.tsp) | 200 · 404 |

## Правила бронирования

| Правило | Как выражено в контракте |
|---------|--------------------------|
| Глобальная занятость: нельзя две записи на одно время (даже разные типы) | Занятые интервалы исключены из выдачи `GET .../slots`; попытка занять занятое время → `409` в `POST /bookings` |
| Окно записи: 14 дней от текущей даты | `GET .../slots` возвращает только слоты окна; попытка брони вне окна / в прошлом → `422` в `POST /bookings` |
| `id` типа события задаёт владелец | `POST /event-types` принимает `id` в теле; повторный `id` → `409` |
| Несуществующий тип события | `404` в `GET /event-types/{id}`, `GET .../slots`, `POST /bookings` |

## Итог

Все сценарии владельца и гостя, а также все правила бронирования имеют отражение в контракте.
Операции разделены по ролям тегами `owner` / `public`. Дополнительные операции (`GET /owner`,
`GET /bookings/{bookingId}`) помечены как supplementary и выходят за рамки обязательных сценариев.
